## ✅ ООП та архітектура

- [x] **Ефективне використання ООП**
  - Класи: Field, Name, Phone, Birthday, Record, AddressBook
  - Спадкування: Phone, Name, Birthday наслідують Field
  - Композиція: Record містить Name, Phone[], Birthday
  - AddressBook містить Record[]

- [x] **Патерни проектування**
  - Strategy Pattern (різні Storage)
  - Factory Pattern (StorageFactory)
  - Decorator Pattern (input_error)

- [x] **SOLID принципи**
  - Single Responsibility: кожен клас має одну відповідальність
  - Open/Closed: легко додати нові команди/storage
  - Liskov Substitution: всі Storage замінюють інтерфейс
  - Interface Segregation: мінімальні інтерфейси
  - Dependency Inversion: залежності через абстракції
