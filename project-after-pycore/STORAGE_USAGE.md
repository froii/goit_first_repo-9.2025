# Використання різних типів збереження

## Архітектура

Використовується патерн **Strategy** + **Factory**:

- `StorageInterface` - абстрактний інтерфейс для всіх типів збереження
- `PickleStorage`, `JSONStorage`, `CSVStorage` - конкретні реалізації
- `StorageFactory` - фабрика для створення потрібного типу збереження

## Особливості форматів

### Pickle
- ✅ Зберігає повну структуру об'єктів Python
- ✅ Найшвидший
- ❌ Не читається людиною
- ❌ Проблеми з версіонуванням

### JSON
- ✅ Читається людиною
- ✅ Кросплатформенний
- ✅ Легко редагувати вручну
- ❌ Потребує серіалізації складних об'єктів

### CSV
- ✅ Відкривається в Excel
- ✅ Простий формат
- ❌ Обмежена структура (тільки плоскі дані)
- ❌ Не підходить для складних об'єктів
